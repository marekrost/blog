We have recently moved all Wordpress-based development in Triton IT to Bedrock by Roots.io. One of the perks of this system is the ability to manage all external code with PHP composer. What is the only caveat?

It's the management of non-public external code or private packages. Particularly paid Wordpress plugins turned out to be annoying. We somehow needed them versioned and managed while not commiting them directly to our own code repository.

While looking for a solution - I have found [Release Belt](https://github.com/Rarst/release-belt), a nice little application that can manage ZIP files as if they were composer packages.

Unfortunately Release Belt was not available as a Docker image and in [Triton IT](https://www.tritonit.cz/) we run all of our key infrastructure in containers. With the assistance of my teammates I have [packaged Release Belt into a docker image](https://github.com/marekrost/docker-release-belt) and made it available directly from [Docker hub](https://hub.docker.com/r/marekrost/release-belt).
